2. What does fast-forward in Git mean?
Ans. a fast forward mean rebasing the code to a branch which is ahead of your current checked out branch state.
3. Why should you have different branches?
Ans. If all developers start making changes in the same branch lets say main  then there are high chances of the code breaking
and conflicts arising. So branches make it easy for the entire product to be worked on seperately merged in the end with out 
any breaks.
4. In a project how many remote Git repositories do you usually have?
In a project there is usually a single repository for each self contained project.
